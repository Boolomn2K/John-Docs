# 工作排期

## 重点关注

1. 没有完整的设计稿，前端不能给出准确工时

2. 前端工程师无法给出排期，给出的是工时

3. 给出的工时是开发时间，不包含测试时间

4. 不要再压力下评估工时

## 四种评估方法

1. 拍脑袋

2. 找有经验同事咨询求平均工时

3. 以往项目经验法

4. 需求分解法

## 要做前端技术方案

**当遇到难度较高或者是没有经验的需求，这时候一定要做技术方案**

这里面内容有

1. 功能拆解

2. 开发工时

3. 核心功能技术方案

4. 测试影响范围

工作时间表和技术方案一般是一起给出

### 工时表排期好处

1. 帮助准确评估工时

2. 有理可依 (扯皮甩锅时挺直脊梁的依据)

3. 预防风险 (遇到倒排工期时的丑话)

4. 降低预期 (预期管理)
